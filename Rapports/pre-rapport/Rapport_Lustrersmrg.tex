class="nb">{</span>Introduction<span class="nb">}</span></div><div class='line' id='LC72'>Lustre est un système de fichiers distribué libre(sous licence GPL v2), utilisé pour de très grandes grappes de serveurs. Son nom vient de la combinaison de deux mots, «Linux» et «cluster». Le projet a commencé comme un projet de recherche en 1999 par Peter Braam, en 2007 Sun Microsystems a continué son développement, en 2010 avec l&#39;acquisition de Sun, a commencé à manternir et sortir nouvelles versions de Lustre, en décembre du 2010 Oracle a arrêté le développement du système de fichiers version 2.x, plusieurs nouvelles organisations surgirent à fournir un soutien et de développement dans un modèle de développement communautaire ouvert, comprenant <span class="k">\textit</span><span class="nb">{</span>Whamcloud<span class="nb">}</span>, <span class="k">\textit</span><span class="nb">{</span>Open Scalable File Systes, Inc. (OpenSFS)<span class="nb">}</span>, <span class="k">\textit</span><span class="nb">{</span>EUROPEAN Open FileSystems (EOFS)<span class="nb">}</span> et autres. En 2012 WhamCloud a été rachetée par Intel.<span class="k">\\</span></div><div class='line' id='LC73'><br/></div><div class='line' id='LC74'><br/></div><div class='line' id='LC75'>Lustre a la possibilité de accueillir des dizaines de milliers de systèmes client et une grande quantité de données, pétaoctets<span class="k">\footnote</span><span class="nb">{</span>Un pétaoctet contient <span class="k">\begin</span><span class="nb">{</span>math<span class="nb">}</span>10<span class="nb">^{</span>15<span class="nb">}</span><span class="k">\end</span><span class="nb">{</span>math<span class="nb">}</span> octets<span class="nb">}</span> de stockage et des centaines de gigaoctets(Go) de débit Entrée/Sortie. </div><div class='line' id='LC76'><br/></div><div class='line' id='LC77'><span class="k">\subsubsection</span><span class="nb">{</span>Les composants de Lustre<span class="nb">}</span></div><div class='line' id='LC78'><span class="k">\begin</span><span class="nb">{</span>description<span class="nb">}</span></div><div class='line' id='LC79'><span class="k">\item</span><span class="na">[Serveur de Métadonnées \textit{(Metadata Server «MDS»)}:]</span></div><div class='line' id='LC80'>Le serveur MDS rend les métadonnées, stockées dans un ou plusieurs MDT, disponibles pour des clients Lustre. Chaque MDS gère les noms et répertoires dans le systèmes de fichiers Lustre.</div><div class='line' id='LC81'><span class="k">\item</span><span class="na">[Cible de Métadonnées \textit{(Metadata Target «MDT»)}:]</span> Le MDT stocke les métadonnées(tels que les noms de fichiers, les répertoires et la structure des fichiers) sur un MDS. Chaque système de fichiers possède un MDT. Une MDT sur une cible de stockage partagé peut être disponible pour de nombreux MDS, bien que l&#39;on devrait utiliser effectivement. Si une MDS actif échoue, un MDS passif peut servir le MDT et le rendre disponible aux clients. Ceci est appelé <span class="k">\textit</span><span class="nb">{</span>MDS failover<span class="nb">}</span>.</div><div class='line' id='LC82'><span class="k">\item</span><span class="na">[Serveur de Stockage d&#39;Objets \textit{(Object Storage Server «OSS»)}:]</span></div><div class='line' id='LC83'>L&#39;OSS fournit un servicce d&#39;Entrée/Sortie de fichiers et <span class="k">\textit</span><span class="nb">{</span>network request handling<span class="nb">}</span> pour un ou plusieurs OST locaux. Typiquement, un OSS sert entre 2 et 8 OST, jusqu&#39;à 8 To<span class="k">\footnote</span><span class="nb">{</span>Un téraoctet contient <span class="k">\begin</span><span class="nb">{</span>math<span class="nb">}</span>10<span class="nb">^{</span>12<span class="nb">}</span><span class="k">\end</span><span class="nb">{</span>math<span class="nb">}</span>octets.<span class="nb">}</span></div><div class='line' id='LC84'><span class="k">\item</span><span class="na">[Cible de Stockage d&#39;Objets \textit{(Object Storage Target «OST»)}:]</span></div><div class='line' id='LC85'>L&#39;OST stocke les données(des morceaux de fichiers de l&#39;utilisateur) comme des objets de données sur un ou plusieurs OSS. Un seul système de fichiers Lustre peut avoir plusieurs OST, servant chacun un sous-ensamble de fichiers données. Il n&#39;est pas nécessairement une correspondance 1:1 entre un fichier et un OST. Pour optimiser les performances, un fichiers peut être distribué sur plusieurs OST.</div><div class='line' id='LC86'><span class="k">\item</span><span class="na">[Serveur de Gestion \textit{(Management Server «MGS»)}:]</span></div><div class='line' id='LC87'>Le MGS stocke les informations de configuration pour tous les systèmes de fichiers Lustre dans une grappe de serveurs. Chaque component Lustre contacte le MGS pour fournir information, et les Lustre clients contactent le MGS pour obtenir information.</div><div class='line' id='LC88'><span class="k">\item</span><span class="na">[Lustre clients:]</span>Lustre clients sont noeuds éxecutant le logiciel Lustre qui leur permet de monter le système de fichiers Lustre.</div><div class='line' id='LC89'><br/></div><div class='line' id='LC90'>Le logiciel client de Lustre est composé d&#39;une interfacce entre le <span class="k">\textit</span><span class="nb">{</span>Linux Virtual File System<span class="nb">}</span> et les serveurs Lustre. Chaque cible(composant de Lustre) a une contrapartie client: Client de Métadonnées (MDC), Client de Stockage d&#39;Objets(OSC) et un Client de Gestion(MGC). Un groupe de OSC sont intégrés dans un Volume d&#39;objets logique(<span class="k">\textit</span><span class="nb">{</span>Logical Object Volume «LOV»<span class="nb">}</span>). Travaillant en collaboration, les OSC fournissent un accès transparent au système de fichiers.</div><div class='line' id='LC91'><br/></div><div class='line' id='LC92'>Les clients qui montent le système de fichiers Lustre, voient un seul, cohérente, espace de noms(<span class="k">\textit</span><span class="nb">{</span>namespace<span class="nb">}</span>), synchronisé toujours. Différents clients peuvent écrire aux différentes parties d&#39;un même fichier en même temps, tandis que d&#39;autres clients peuvent lire le fichier.</div><div class='line' id='LC93'><span class="k">\end</span><span class="nb">{</span>description<span class="nb">}</span></div><div class='line' id='LC94'><span class="k">\subsubsection</span><span class="nb">{</span>Principales Caractéristiques de Lustre<span class="nb">}</span></div><div class='line' id='LC95'><br/></div><div class='line' id='LC96'>Les principales caractéristiques de Lustre comprennent: <span class="k">\\</span></div><div class='line' id='LC97'><br/></div><div class='line' id='LC98'><span class="k">\begin</span><span class="nb">{</span>description<span class="nb">}</span></div><div class='line' id='LC99'><br/></div><div class='line' id='LC100'><span class="k">\item</span><span class="na">[Évolutivité: ]</span></div><div class='line' id='LC101'>&nbsp;Lustre échelles haut ou le bas par rapport au nombre de postes clients, stockage sur disque et bande passante. Actuellement , Lustre est en cours d&#39;exécution dans des environnements de production avec un maximum de 26 000 postes clients, avec de nombreux grappes d&#39;entre 10,000 et 20,000 clients. Dans un environnement de test, une performance de 130 Go par seconde a étée soutenue. Lustre seul débit de poste client a été mesurée à 2 Go par seconde (max) et OSS débit de 2,5 Go par seconde (max). Lustre a été exécuté à 240 Go par seconde sur le système de fichiers d&#39;araignée(Spider File System) à Oak Ridge National Laboratories.</div><div class='line' id='LC104'><br/></div><div class='line' id='LC105'><span class="k">\item</span><span class="na">[Conformité POSIX:]</span>Dans un cluster, la conformité POSIX signifie que la plupart des opérations sont atomiques et les clients ne voient jamais des données périmées ou métadonnées .</div><div class='line' id='LC106'><span class="k">\item</span><span class="na">[Haute Disponibilité:]</span>Lustre propose des partitions de stockage partagés pour les cibles de l&#39;OSS (OST), et une partition de stockage partagé pour la cible MDS (MDT).</div><div class='line' id='LC107'><br/></div><div class='line' id='LC108'><span class="k">\item</span><span class="na">[Sécurité:]</span>En Lustre, il s&#39;agit d&#39;une option pour les connexions TCP seulement de ports privilégiés. Manipulation des membres du groupe est basée sur le serveur. POSIX listes de contrôle d&#39;accès (ACL) sont pris en charge .</div><div class='line' id='LC109'><span class="k">\item</span><span class="na">[Open Source:]</span> Lustre est sous licence GNU GPL v2.</div><div class='line' id='LC110'><span class="k">\item</span><span class="na">[]</span></div><div class='line' id='LC111'><span class="k">\end</span><span class="nb">{</span>description<span class="nb">}</span></div><div class='line' id='LC112'><span class="k">\subsubsection</span><span class="nb">{</span>Lustre Network<span class="nb">}</span></div><div class='line' id='LC113'><br/></div><div class='line' id='LC114'><span class="k">\textit</span><span class="nb">{</span>Lustre Network (LNET)<span class="nb">}</span> est une API de réseau personnalisé qui fournit l&#39;infrastructure de communication qui gère l&#39;Entrée/Sortie des métadonnées et fichiers de données pour les serveurs et les clients du système de fichiers Lustre.</div><div class='line' id='LC115'><br/></div><div class='line' id='LC116'>LNET prend en charge plusieurs types de réseaux couramment utilisés, tels que les réseaux <span class="k">\textit</span><span class="nb">{</span>InfiniBand<span class="nb">}</span> et <span class="k">\textit</span><span class="nb">{</span>IP<span class="nb">}</span>, et permet la disponibilité simultanée sur plusieurs types de réseaux avec routage entre eux. Accès à la mémoire direct à distance (RDMA) est autorisée lorsqu&#39;elle est soutenue par des réseaux sous-jacentes utilisant le pilote réseau Lustre approprié (<span class="k">\textit</span><span class="nb">{</span>Lustre Network Driver «LND»<span class="nb">}</span>). LNDs sont chargés dans la pile de pilotes, avec une LND pour chaque type de réseau en cours d&#39;utilisation.</div><div class='line' id='LC118'><br/></div><div class='line' id='LC119'><span class="k">\subsubsection</span><span class="nb">{</span>Réseaux Lustre<span class="nb">}</span></div><div class='line' id='LC120'><br/></div><div class='line' id='LC121'>Un réseau Lustre est composé de clients et serveurs exécutant le logiciel Lustre. Il n&#39;a pas besoin d&#39; être limité à un sous-réseau de LNET mais peut s&#39;étendre sur plusieurs réseaux, le routage est possible entre les réseaux. D&#39;une manière similaire, un réseau unique peut avoir plusieurs sous-réseaux LNET.</div><div class='line' id='LC122'><br/></div><div class='line' id='LC123'><span class="k">\subsubsection</span><span class="nb">{</span>Lustre cluster<span class="nb">}</span></div><div class='line' id='LC124'><br/></div><div class='line' id='LC125'><br/></div><div class='line' id='LC126'><span class="k">\begin</span><span class="nb">{</span>figure<span class="nb">}</span>[Cluster]</div><div class='line' id='LC127'><span class="k">\center</span><span class="nb">{</span><span class="k">\includegraphics</span><span class="na">[width=1.0\linewidth]</span><span class="nb">{</span>scaled<span class="nb">_</span>cluster<span class="nb">}}</span></div><div class='line' id='LC128'><span class="k">\caption</span><span class="nb">{</span>Scaled Cluster<span class="nb">}</span></div><div class='line' id='LC129'><span class="k">\label</span><span class="nb">{</span>scaled<span class="nb">_</span>cluster<span class="nb">}</span></div><div class='line' id='LC130'><span class="k">\end</span><span class="nb">{</span>figure<span class="nb">}</span></div><div class='line' id='LC131'><br/></div><div class='line' id='LC132'>À grande échelle, un cluster de système de fichiers Lustre peut inclure des centaines de OSS et des milliers de clients, Figure <span class="k">\ref</span><span class="nb">{</span>scaled<span class="nb">_</span>cluster<span class="nb">}</span>, Plus d&#39;un type de réseau peut être utilisé dans un cluster Lustre. Si le fichier MDT pointe à un objet, toutes les données de fichier sont stockées dans cet objet.
